# InputParser 구조체(시발 수정함 구조체 아님 객체임)

## 목적

- 클라이언트에서 각 입력이 들어왔을 때 각각의 동작을 어떻게 해야할지
맨앞의 식별자를 따라 판별하고
최종적으로 `SendPacket()`으로 보내야 할
패킷의 정보를 반환해야할 필요가 있었음.
  - `ParsedInput`
    - `PacketType` `type`
    - `const char*` `payload`
    - `uint32_t` `length`

> 결국, 입력받은 메시지를 실행할 동작 / 보낼 패킷 헤더에 따라 분류해서 실행해야할 필요가 있었기 때문.

> 이거 닉네임 시스템 만들다가 닉네임 수정 기능 만들려는데
> 도저히 기존 구조로는 닉네임 설정 메시지와 일반적인 메시지를 구분 못해서 필요성을 느낀거임.

## 설계

### 일단 어떻게 입력에 따른 각각의 동작을 정의를 해야할까?

이런 방식밖에 안 떠오름

(`입력 메시지 식별자` 설계)

- (아무것도 없음, 즉, /으로 시작하지 않음.) = 일반 메시지
- "/nick "= 닉네임 설정
- "/quit " = 클라이언트 종료

맨 앞의 문자열을 검사하고 그게 식별자랑 일치하면 각각의 동작으로 처리.

### 파싱 함수(`InputParser::Parse()`)

입력받은 문자열의 맨 앞에 있는 입력 메시지 식별자를 보고 파싱해야함.

- 일단 `ParsedInput` 구조체를 반환해야함
- 내부에서 `ParsedInput` 구조체를 생성한 후 각각의 입력 메시지 식별자에 맞게
`type` / `payload` / `length`를 설정 후 반환하는 식?

더럽게 어렵네 시발

#### 동작 순서

1. 사용자로부터 입력받은 `std::string`을 입력받음.
2. 반환하기 위한 `ParsedInput` 구조체 생성
3. 사용자에게서 입력받은 문자열의 맨 앞 글자를 인식해서 `/`이면 4번 과정, 아니라면 일반메시지로 처리
3-1. 일반 메시지로 처리할 때는 
그저 `ParsedInput` 구조체의 
`type`을 `PacketType::CHAT_MESSAGE`로, 
`length`를 해당 문자열의 길이로, 
`payload`는 그냥 해당 문자열을 참조. 
그 후 종료.
4. 사용자에게서 입력받은 문자열의 두 번째 글자부터 인식해서 각 식별자에 맞는 처리. 추후에 다른 식별자를 추가할 수 있음.
5. 그 후 처리된 `ParsedInput`을 반환

### 입력 메시지 식별자를 별도의 공간에 모아놓자.

`ClientMessageType`으로 따로 구조체를 만들어서 별도의 공간에..

- `const std::string` `normal_message`(일반 메시지) = (이건 어케하냐)
- `const std::string` `change_nickname`(닉네임 변경) = `"/nick "`
- `const std::string` `quit_client`(클라이언트 종료) = `"/quit"` (뒤에 추가적인 문자 없음.)

대충 이런 식으로 모아놓자.
유효하지 않은 명령어도 검사해야할 듯.

일단 일반 메시지 / 그 외를 판별하기 위해
파싱 함수(`Parse()`)에서 `/`를 검사한 후 그 후에 뭐가 오는지(`nick `, `quit ` 등)를 검사하는 식으로..

### `struct` `ParsedInput`(파싱한 결과를 담을 구조체)

#### 실제 구현 예시

```cpp
// 입력받은 메시지를 파싱한 결과
struct ParsedInput {
    PacketType type; // 오직 보내야할 패킷 타입만을 나타냄
    uint32_t length; // 보내야할 길이를 나타냄
    std::string payload; // 실제로 보낼 메시지를 나타냄 (/nick같은 메시지 식별자 절삭한)

    bool quit = false; // 입력받은 메시지가 종료 메시지냐
    bool valid = true; // 입력받은 메시지가 유효하냐
};
```

### 입력 -> 동작 정책

입력:

- "/quit"        -> quit = true
- "/nick name"   -> type = NICKNAME_CHANGE, payload = "name"
- "/(unknown)"   -> valid = false
- "hello"        -> type = CHAT_MESSAGE, payload = "hello"

### 입력 후 송신까지의 과정

1. 메시지를 입력받는다.
2. `ParsedInput::Parse()` 함수로 해당 메시지를 어떻게 송신 / 처리해야하는지를 얻는다.
3. `ClientApp::SendPacket()` 함수로 해당 메시지를 `ParsedInput`에 맞춰 송신하거나,
`/quit`같이 `SendPacket()`을 사용하지 않는 메시지는 기타 처리를 한다.

### `Parse()` 함수 밖에서는 어떻게 처리해야할까?

- 반환된 `ParsedInput`의 type / quit / valid 필드를 보고 각 상황에 맞는 처리를 한다.
- 무조건 valid 검사 -> quit 검사 -> type 검사 순으로 해야 유효하다.
  - 일단 입력의 유효성이 검사되어야 뒤의 작업들을 처리할 수 있다.
  - 그 후, 입력이 quit인지를 먼저 검사해야 type만 검사해서는 알 수 없는 종료 여부를 알 수있다.
  - 마지막으로, 다른 검사들을 다 통과해야만 type만 검사해서 알 수 있는 것들만 남으므로 type 검사를 한다.
